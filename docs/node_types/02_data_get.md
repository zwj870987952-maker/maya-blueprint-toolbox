# 类型 2：数据获取

## 声明

数据获取节点负责从 Maya 场景或当前 Maya 环境里读取数据。

它可以读取 Maya。
它不能修改 Maya。
它不能改变选择、创建节点、删除节点、重命名节点、导入文件或导出文件。

这一类节点的设计优先跟随 Maya 用户的实际操作习惯。用户通常想要的是“当前选择”“场景里所有骨骼”“这个模型的蒙皮”“这个物体的材质”，而不是先手动拆 DAG、DG、Shape、History、Connection。

底层实现必须区分 DAG、DG 和属性连接关系，但公开节点优先做成快捷入口。

## 节点颜色

`#38C7C7`

## 允许行为

- 读取当前选择
- 根据节点名称得到 Maya 节点列表
- 根据 Maya 类型得到节点列表
- 从模型或节点反查相关 SkinCluster
- 从模型或节点反查相关 BlendShape
- 从模型或节点反查相关变形器
- 从模型或节点反查相关材质
- 从模型或节点反查相关约束
- 读取属性引用的当前值
- 检查属性引用的类型、锁定、可写、连接等信息

## 禁止行为

- 不能选择节点
- 不能设置属性
- 不能创建属性
- 不能连接或断开属性
- 不能创建、删除、重命名对象
- 不能导入或导出文件
- 不能改变时间轴、单位、命名空间、工程路径等 Maya 环境状态

## 通用端口规则

输入端口按查询来源决定。

输出端口必须是读取结果，不能输出执行副作用。

常用输出类型：

| 输出内容 | 端口类型 |
| --- | --- |
| Maya 节点列表 | `NODE_LIST` |
| Maya 属性引用列表 | `ATTR_LIST` |
| 属性值 | `VALUE` |
| 检查报告 | `STRING` |

## 当前公开节点

| 节点 | 输入端口 | 输出端口 | 说明 |
| --- | --- | --- | --- |
| 当前选择 | 无 | `节点: NODE_LIST` | 获取 Maya 当前选择 |
| 节点名称 | 无 | `节点: NODE_LIST` | 录入 Maya 当前选择或手动添加节点名，输出节点列表 |
| 按 Maya 类型获取节点 | 无 | `节点: NODE_LIST` | 获取 joint、mesh、skinCluster、blendShape、约束、材质等 Maya 类型 |
| 获取 SkinCluster | `节点: NODE_LIST` | `节点: NODE_LIST` | 从模型、Shape 或相关节点找到蒙皮节点 |
| 获取 BlendShape | `节点: NODE_LIST` | `节点: NODE_LIST` | 从模型、Shape 或相关节点找到 BlendShape 节点 |
| 获取变形器 | `节点: NODE_LIST` | `节点: NODE_LIST` | 从模型、Shape 或相关节点找到常见变形器 |
| 获取材质 | `节点: NODE_LIST` | `材质节点: NODE_LIST` | 从模型、Shape 找到材质节点 |
| 获取约束 | `节点: NODE_LIST` | `约束节点: NODE_LIST` | 从被约束或驱动对象找到相关约束 |
| 获取属性 | `属性引用: ATTR_LIST` | `值: VALUE` | 读取属性当前值 |
| 检查属性 | `属性引用: ATTR_LIST` | `值: VALUE`、`报告: STRING` | 读取属性值和属性状态 |

## 录入交互规则

数据获取节点不能只依赖手写字符串。只要数据来源可以从 Maya 当前环境读取，就应该提供录入按钮。

`节点名称` 节点必须支持：

- 点击按钮录入 Maya 当前选择
- 多个节点累计录入
- 手动补充节点名
- 删除选中的已录入节点
- 保存 / 加载时保留录入列表

属性引用相关节点必须支持：

- 从内置常用属性列表里搜索并添加属性名
- 从 Maya 当前选择刷新可用属性名
- 从 Channel Box 录入当前选中的属性名，不保存对象名
- 多个属性累计录入
- 删除选中的已录入属性
- 保存 / 加载时保留录入列表

属性录入结果应该是可复用的属性名，例如 `translateX`，而不是 `pCube1.translateX`。

即使用户在 Maya 里选择的是 `pCube1.translateX`，录入列表里也只显示并保存 `translateX`。运行时由输入节点决定作用对象：

```text
输入节点 + 属性名 -> 属性引用列表
```

## 暂不公开为独立节点的底层查询

这些关系真实存在，但第一版不做成独立公开节点，避免画布过碎：

| 底层查询 | 处理方式 |
| --- | --- |
| 获取 Shape | 作为获取 SkinCluster、BlendShape、变形器、材质时的内部步骤 |
| 获取历史节点 | 不做公开节点；只在快捷查询内部用于找到变形器 |
| 获取输入连接 | 暂不公开；以后如果做材质网络、动画曲线网络，再单独定义 |
| 获取输出连接 | 暂不公开；以后如果做依赖网络分析，再单独定义 |
| 获取 DAG 父子层级 | 暂不公开；以后放到“数据转换”或单独“层级处理”类里讨论 |

## Maya 关系规则

Maya 里至少要区分三种关系：

| 关系 | 说明 | 常用命令 |
| --- | --- | --- |
| DAG 层级 | Outliner 里的父子空间层级，例如 transform 下面有 shape，joint 有父子层级 | `cmds.listRelatives` |
| DG 依赖 | 节点计算关系，例如 skinCluster、blendShape、animCurve、材质网络 | `cmds.listHistory`、`cmds.listConnections` |
| 属性引用 | 具体到 `node.attribute` 的 plug，例如 `pCube1.translateX` | `cmds.objExists`、`cmds.getAttr`、`cmds.attributeQuery` |

公开节点命名不能笼统使用“物体”。

优先使用：

- 节点
- Transform
- Shape
- DAG 节点
- DG 节点
- 属性引用

## 设计备注

“获取 Shape”本身有实际价值，比如材质、蒙皮和变形器通常挂在 shape 或 shape 的历史/连接上。但是对用户来说，他们更常从 transform 模型开始操作。

所以公开节点应该让用户可以这样连：

```text
当前选择 -> 获取 SkinCluster -> 后续操作
当前选择 -> 获取材质 -> 后续操作
当前选择 -> 获取 BlendShape -> 后续操作
```

而不是要求用户这样连：

```text
当前选择 -> 获取 Shape -> 获取历史节点 -> 过滤 SkinCluster
```

底层可以做后一套逻辑，但画布上优先展示前一套快捷逻辑。
