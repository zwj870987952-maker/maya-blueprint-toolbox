# Maya 蓝图工具盒节点类型规则

本文档定义所有节点必须遵守的通用规则。新增节点前，先判断它属于哪个“功能类型”，再写节点规格和执行逻辑。

## 总原则

- 节点分类按 Maya 实际操作习惯制定，而不是按 Maya 菜单或代码文件分类。
- 优先提供符合日常操作习惯的快捷节点，避免让用户频繁拼接底层 DAG / DG 查询。
- 底层 DAG / DG / 属性连接关系要在实现里分清楚，但不一定全部暴露成公开节点。
- 节点头部颜色表示“节点功能类型”。
- 端口颜色表示“端口数据类型”。
- 连线颜色继承输出端口颜色。

## 功能类型颜色

| 功能类型 | 颜色 | 说明 |
| --- | --- | --- |
| 常量 | `#4EA1FF` | 只提供固定值，不读取 Maya，不修改 Maya |
| 数据获取 | `#38C7C7` | 从 Maya 或环境读取数据，不修改 Maya |
| 数据转换 | `#9B6DFF` | 整理、过滤、组合输入数据，不直接读取或修改 Maya |
| Maya 操作 | `#F2994A` | 会修改 Maya 场景、选择、文件或外部结果 |
| 流程控制 | `#F2C94C` | 控制执行顺序、条件、循环 |
| 调试 / 报告 | `#7FB069` | 输出、检查、报告、预览数据 |

## 端口类型颜色

| 端口类型 | 颜色 | 说明 |
| --- | --- | --- |
| `ANY` | `#BFC7D5` | 调试用通用输入，优先只用于调试 / 报告节点 |
| `STRING` | `#56CCF2` | 文本 |
| `NUMBER` | `#BB6BD9` | 数字 |
| `BOOL` | `#F2994A` | 布尔 |
| `PATH` | `#6FCF97` | 文件路径 |
| `NODE_LIST` | `#2D9CDB` | Maya 节点列表，可包含 DAG 或 DG 节点 |
| `ATTR_LIST` | `#F2994A` | Maya 属性引用列表 |
| `VALUE` | `#D0A8FF` | 通用属性值或计算值 |
| `FILE_RESULT` | `#27AE60` | 文件操作结果 |

## 新建节点检查表

新增节点必须回答这些问题：

1. 它属于哪个功能类型？
2. 它是否读取 Maya？
3. 它是否修改 Maya？
4. 它是否依赖当前选择？
5. 它输入端口是什么？
6. 它输出端口是什么？
7. 它是否可以用 Maya 官方 `maya.cmds` 实现？
8. 它是用户常用快捷节点，还是底层内部工具？

如果节点会修改 Maya，就不能放在“数据获取”或“数据转换”里。

如果节点只是把输入数据换成另一种格式，就优先放在“数据转换”里。

如果节点是为了减少 Maya 常用操作步骤，可以作为快捷节点公开；底层实现可以组合 `cmds.listRelatives`、`cmds.listHistory`、`cmds.listConnections` 等官方命令。
