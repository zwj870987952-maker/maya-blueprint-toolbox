# 工具拆解：复制粘贴世界坐标 v4

源工具：

```text
C:\BaiduSyncdisk\自定义脚本\py\py3\复制粘贴世界坐标v4.txt
```

## 抽象方式

这个工具不应该被拆成很多零碎命令节点，而应该抽象成：

```text
通用信息节点 + 核心功能节点
```

核心功能节点只负责动画师真正想做的事：

1. 复制帧：记录某些对象在某些帧上的世界空间变换数据。

是否生成 JSON、是否继续粘贴到其他对象属性，不再拆成另一个核心功能节点，而由“复制帧”的输出口连接状态决定。

对象来源、帧范围、当前帧、粘贴属性通道、逐帧或只关键帧，这些都不应该写死在核心节点里，而应该通过端口连接进来。

## 核心功能

| 核心功能 | 动画师语言 | 节点 |
| --- | --- | --- |
| 复制帧 | 把这些对象在这些帧的世界变换数据复制下来；输出没连接就保存 JSON，输出连接了就把数据送去粘贴 | 复制帧 |

## 核心节点规格

### 复制帧

| 字段 | 规格 |
| --- | --- |
| 类型 ID | `maya.animation.copy_frame` |
| 功能类型 | 数据获取 / 条件文件输出 |
| 输入 | `复制对象: NODE_LIST`、`帧: FRAME_LIST`、`通道: CHANNEL_LIST` |
| 输出 | `帧数据: TRANSFORM_FRAME_DATA` |
| 参数 | `记录短名称: BOOL=True`、`未连接时保存 JSON: BOOL=True`、`JSON 路径: PATH` |
| 作用 | 读取复制对象在指定帧上的世界变换数据。默认识别并记录对象全部变换通道。`通道` 输入表示粘贴时允许写入的属性通道。 |

输出行为：

| 输出口状态 | 行为 |
| --- | --- |
| `帧数据` 没有连接 | 生成包含对象、帧、世界变换数值、通道信息的 JSON 文件。 |
| `帧数据` 已连接 | 不强制生成 JSON，把复制到的帧数据传给连接目标，由连接目标把数据粘贴到对应对象属性。 |

`帧数据` 应包含：

| 字段 | 含义 |
| --- | --- |
| `source_nodes` | 复制对象列表。 |
| `frames` | 复制帧列表。 |
| `recorded_channels` | 实际记录的变换通道，默认包含全部 transform 通道。 |
| `paste_channels` | 允许粘贴的属性通道，由 `通道: CHANNEL_LIST` 输入决定。 |
| `samples` | 每个对象、每一帧的世界变换数值。 |
| `short_name` | 可选短名称，用于跨层级或跨命名空间匹配。 |

## 通用输入信息节点

这些节点不是这个工具独有，后续手脚锁定、道具跟随、空间切换、烘焙工具都能复用。

### 对象来源

| 节点 | 类型 | 输出 | 作用 |
| --- | --- | --- | --- |
| 当前选择 | 数据获取 | `节点: NODE_LIST` | 用当前选择作为复制对象或粘贴对象。 |

| 取节点列表项 | 数据转换 | `节点: NODE_LIST` | 单对象复制 / 单对象粘贴时使用。 |
| 按层级排序节点 | 数据转换 | `节点: NODE_LIST` | 多父子对象粘贴时父级优先。 |

### 通道来源

| 节点 | 类型 | 输出 | 作用 |
| --- | --- | --- | --- |
| 获取通道栏选择 | 数据获取 | `通道: CHANNEL_LIST` | 读取 Channel Box 选中的 translate / rotate / scale 通道。 |
| 变换通道 | 数据转换 | `通道: CHANNEL_LIST` | 用勾选参数生成位移、旋转、缩放通道。 |
| 合并通道列表 | 数据转换 | `通道: CHANNEL_LIST` | 组合多组通道。 |

### 时间来源

| 节点 | 类型 | 输出 | 作用 |
| --- | --- | --- | --- |
| 当前帧 | 数据获取 | `帧: FRAME_LIST` | 如果输入口没有连接时间值，输出 Maya 当前帧；如果连接了 `时间: NUMBER`，输出指定数值帧。 |
| 范围帧 | 数据获取 | `帧: FRAME_LIST`、`帧范围: FRAME_RANGE` | 如果没有连接起始/结束时间，输出当前时间轴范围；如果连接了起始和结束时间，输出该范围。 |
| 播放范围 | 数据获取 | `帧范围: FRAME_RANGE`、`帧列表: FRAME_LIST` | 使用 playback range。 |
| 获取对象关键帧 | 数据获取 | `帧列表: FRAME_LIST` | 获取对象已有关键帧。 |
| 帧范围过滤 | 数据转换 | `帧列表: FRAME_LIST` | 把关键帧限制在指定范围内。 |
| 逐帧采样 | 数据转换 | `帧范围: FRAME_RANGE` -> `帧列表: FRAME_LIST` | 默认逐帧处理。 |

### 当前帧

| 字段 | 规格 |
| --- | --- |
| 类型 ID | `maya.timeline.current_frame` |
| 功能类型 | 数据获取 |
| 输入 | `时间: NUMBER`，可选 |
| 输出 | `帧: FRAME_LIST` |
| 默认行为 | 输入没连接时读取 Maya 当前帧；输入连接时输出该数值帧。 |

### 范围帧

| 字段 | 规格 |
| --- | --- |
| 类型 ID | `maya.timeline.frame_range` |
| 功能类型 | 数据获取 |
| 输入 | `起始帧: NUMBER`，可选；`结束帧: NUMBER`，可选 |
| 输出 | `帧范围: FRAME_RANGE`、`帧: FRAME_LIST` |
| 默认行为 | 起始/结束都没连接时读取当前时间轴范围；连接后使用输入数值范围。 |

## 端口如何决定工具行为

### 复制当前选择当前帧，输出未连接时保存 JSON

```text
当前选择 -> 复制对象
当前帧 -> 复制帧
变换通道 -> 通道
复制帧
```

### 复制指定范围帧，输出连接时继续粘贴

```text
节点名称 -> 复制对象
范围帧 -> 帧
变换通道 -> 通道
复制帧.帧数据 -> 连接目标对象属性
```

### 按列表顺序 1 对 1 粘贴

复制对象列表和粘贴对象列表数量必须相同。粘贴属性永远按列表顺序匹配，不按名称或短名称自动猜：

```text
节点名称(复制对象 A, B, C) -> 复制帧.复制对象
范围帧 / 当前帧 -> 复制帧.帧
变换通道 / 获取通道栏选择 -> 复制帧.通道

节点名称(粘贴对象 1, 2, 3) -> 创建属性引用.节点
创建属性引用(translateX...rotateZ) -> 设置属性.属性引用
复制帧.帧数据 -> 设置属性.值
```

结果是：

```text
A -> 1
B -> 2
C -> 3
```

如果要改变对应关系，不改功能节点规则，而是在复制对象输入端或粘贴对象输入端插入排序节点：

```text
节点名称(复制对象 A, B, C)
-> 按名称排序节点 / 反转节点列表 / 按层级排序节点
-> 复制帧.复制对象

节点名称(粘贴对象 1, 2, 3)
-> 按名称排序节点 / 反转节点列表 / 按层级排序节点
-> 创建属性引用.节点
```

例如目标端反转后：

```text
复制对象: A, B, C
粘贴对象: 3, 2, 1

A -> 3
B -> 2
C -> 1
```

### 默认逐帧处理

```text
范围帧 -> 帧
复制帧
```

### 只处理关键帧

```text
复制对象 -> 获取对象关键帧
范围帧 -> 帧范围过滤
帧范围过滤 -> 复制帧.帧
```

## 原工具参数归属

| 原工具参数 / 逻辑 | 蓝图归属 |
| --- | --- |
| 选中对象 | `当前选择` 节点输出，接到 `复制帧.复制对象`。 |
| 位移 / 旋转通道判断 | `获取通道栏选择` 或 `变换通道` 节点输出，接到 `复制帧.通道`，作为粘贴属性通道。 |
| 时间滑块范围 | `范围帧` 节点默认行为。 |
| 当前帧 | `当前帧` 节点默认行为。 |
| 逐帧处理 | `范围帧` 输出帧列表后直接接入 `复制帧.帧`。 |
| 只处理关键帧 | `获取对象关键帧` + `帧范围过滤` 节点组合。 |
| 世界坐标数值 | `复制帧` 的 `TRANSFORM_FRAME_DATA` 输出。 |
| 临时 JSON 文件 | `复制帧` 输出口未连接时自动生成 JSON。 |
| 粘贴到对象属性 | `复制帧.帧数据` 输出口连接到目标对象属性时触发。 |
| 复制/粘贴对象对应关系 | 固定按列表顺序 1 对 1。需要调整对应关系时，在输入端使用排序节点。 |

## 新端口类型建议

| 端口类型 | 数据含义 |
| --- | --- |
| `FRAME_RANGE` | 帧范围。 |
| `FRAME_LIST` | 明确帧列表。 |
| `CHANNEL_LIST` | 通道列表，例如 translateX、rotateY。 |
| `TRANSFORM_FRAME_DATA` | 带对象、帧、通道、世界坐标数值的复制结果。 |
| `ANIM_REPORT` | 粘贴结果、失败对象、误差和处理统计。 |
