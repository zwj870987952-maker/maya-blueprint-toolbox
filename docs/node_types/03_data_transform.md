# 类型 3：数据转换

## 声明

数据转换节点只整理输入数据。

它不读取 Maya 场景。
它不修改 Maya 场景。
它不依赖当前选择。
它不能调用 `cmds.ls`、`cmds.nodeType`、`cmds.listConnections`、`cmds.listHistory` 这类 Maya 查询命令。

数据转换节点的作用是把“数据获取”节点输出的结果整理成后续“Maya 操作”节点需要的格式。

列表类转换逻辑可以通用于节点列表和属性引用列表，但第一版不做泛型端口。为了保持连接类型安全，公开节点按数据类型成对提供。

## 节点颜色

`#9B6DFF`

## 当前节点

| 节点 | 输入端口 | 输出端口 | 说明 |
| --- | --- | --- | --- |
| 创建属性引用 | `节点: NODE_LIST` | `属性引用: ATTR_LIST` | 节点列表 + 属性名列表 -> 属性引用列表 |
| 合并节点列表 | `节点 A: NODE_LIST`、`节点 B: NODE_LIST` | `节点: NODE_LIST` | 合并两个节点列表，不去重 |
| 去重节点列表 | `节点: NODE_LIST` | `节点: NODE_LIST` | 删除重复节点，保留原顺序 |
| 按名称排序节点 | `节点: NODE_LIST` | `节点: NODE_LIST` | 按短名称或完整路径文本排序，不查询 Maya |
| 反转节点列表 | `节点: NODE_LIST` | `节点: NODE_LIST` | 反转节点列表顺序 |
| 按层级排序节点 | `节点: NODE_LIST` | `节点: NODE_LIST` | 按 DAG 路径文本深度排序，父级优先或子级优先 |
| 按名称过滤节点 | `节点: NODE_LIST`、`名称文本: STRING` | `匹配节点: NODE_LIST`、`未匹配节点: NODE_LIST` | 只按节点名文本过滤，不查询 Maya 类型 |

| 合并属性引用列表 | `属性 A: ATTR_LIST`、`属性 B: ATTR_LIST` | `属性引用: ATTR_LIST` | 合并两个属性引用列表，不去重 |

| 属性引用列表数量 | `属性引用: ATTR_LIST` | `数量: NUMBER` | 输出属性引用列表长度 |

## 设计规则

- 如果节点需要从 Maya 场景里查询类型、历史、连接或存在性，它不属于数据转换。
- 如果节点只处理已经传入的节点名、属性引用、文本、数字、路径，它优先属于数据转换。
- 数据转换节点可以改变数据结构，但不能改变 Maya 场景状态。
- 单项输出仍尽量保持列表类型，例如 `取节点列表项` 输出 `NODE_LIST`，方便继续连接已有 Maya 操作节点。
- 节点排序只按传入的节点名字符串处理，不验证节点是否存在，也不查询 Maya 层级。
- 节点列表和属性引用列表的转换逻辑可以共用代码，但公开端口必须保持明确类型。
- `创建属性引用` 是专用转换节点，不属于通用列表处理。

## 不放在本类的例子

| 功能 | 原因 |
| --- | --- |
| 按 Maya 类型过滤节点 | 需要查询 `cmds.nodeType`，属于数据获取 |
| 获取 Shape | 需要查询 DAG，属于数据获取或内部快捷查询 |
| 获取 SkinCluster | 需要查询历史 / 连接，属于数据获取 |
| 选择节点 | 会修改 Maya 当前选择，属于 Maya 操作 |
