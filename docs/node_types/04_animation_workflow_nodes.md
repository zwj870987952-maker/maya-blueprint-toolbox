# 类型 4：动画工作流节点规划

## 声明

本文档先记录动画师视角下真正有用的工作流能力，不先从 `maya.cmds` 或 Maya 菜单倒推实现。

动画工作流节点不是单一功能类型。它会横跨：

- 数据获取：读取时间轴、当前帧、关键帧、动画层、约束、通道选择。
- 数据转换：整理帧列表、通道列表、Pose 数据、匹配关系。
- Maya 操作：设置关键帧、粘贴世界变换、移动关键帧、烘焙、创建/删除临时约束。
- 调试 / 报告：检查范围外关键帧、缺失动画、未烘焙约束、动画层风险。

公开分类仍遵守 `docs/node_type_rules.md`。节点归类按“是否读取 Maya / 是否修改 Maya”判断，不按代码模块判断。

## 动画师核心目标

| 目标 | 说明 |
| --- | --- |
| 管理时间 | 明确当前帧、播放范围、时间滑块选择范围、导出范围。 |
| 批量 K 帧 | 对控制器、骨骼或指定属性批量设置关键帧。 |
| 编辑节奏 | 移动、复制、删除、缩放、循环关键帧。 |
| 处理动画曲线 | 切线统一、曲线简化、异常曲线检查。 |
| 保持世界空间 | 手持道具、脚锁地面、角色接触物体、空间切换时保持不跳。 |
| 约束辅助动画 | 临时约束到道具、Locator 或目标控制器，再烘焙回动画。 |
| 烘焙动画 | 把约束、表达式、动画层或控制器结果落成普通关键帧。 |
| 交付前检查 | 检查帧段、范围外 key、动画层、未烘焙约束、命名空间和 root motion。 |

## 建议新增端口类型

现有端口类型可以先用 `VALUE` 承载复杂数据，但动画模块长期应补明确端口类型，避免所有高级节点都变成通用值。

| 端口类型 | 建议颜色 | 数据含义 |
| --- | --- | --- |
| `FRAME_RANGE` | `#F2C94C` | 帧范围，例如 `{start, end, include_end}`。 |
| `FRAME_LIST` | `#F2C94C` | 明确帧列表，例如 `[1, 5, 9, 13]`。 |
| `CHANNEL_LIST` | `#F2994A` | Maya 通道名列表，例如 `translateX/rotateY`。 |
| `TRANSFORM_DATA` | `#D0A8FF` | 世界/局部变换快照，包含节点、位移、旋转、缩放或矩阵。 |
| `POSE_DATA` | `#D0A8FF` | 控制器属性值快照，适合保存/应用 Pose。 |
| `ANIM_REPORT` | `#7FB069` | 动画检查报告，包含问题列表和摘要。 |

## 设计方式

复杂动画工具不应该被拆成一堆互相绑定的小命令，而应该拆成：

| 层级 | 说明 |
| --- | --- |
| 核心功能节点 | 动画师真正想完成的动作，例如复制帧、粘贴帧、烘焙帧、转移动画。 |
| 通用输入节点 | 给核心功能提供对象、帧、通道、匹配关系、采样规则。 |
| 报告节点 | 输出结果、失败对象、检查信息。 |

核心功能节点应该少而稳定。端口连接不同，效果不同。

例如“复制粘贴世界坐标”不应拆成“获取世界坐标、应用世界坐标、设置关键帧”等一串命令节点，也不需要拆成复制和粘贴两个核心节点，而应抽象为：

```text
复制帧
```

复制对象、帧范围、当前帧、粘贴属性通道、逐帧还是关键帧，这些都作为端口输入。输出口没连接时生成 JSON；输出口连接时把帧数据传给连接目标进行粘贴。

复制帧数据粘贴到目标属性时，功能规则固定为列表顺序 1 对 1。需要改变对应关系时，不在功能节点里增加映射模式，而是在复制对象输入端或目标对象输入端使用通用排序节点改变列表顺序。

## 通用端口类型补充

| 端口类型 | 建议颜色 | 数据含义 |
| --- | --- | --- |
| `TRANSFORM_FRAME_DATA` | `#D0A8FF` | 对象在指定帧上的世界/局部变换采样数据。 |

## 第一批核心功能节点

| 节点 | 类型 | 输入 | 输出 | 作用 |
| --- | --- | --- | --- | --- |
| 复制帧 | 数据获取 / 条件文件输出 | `复制对象: NODE_LIST`、`帧: FRAME_LIST`、`通道: CHANNEL_LIST` | `帧数据: TRANSFORM_FRAME_DATA` | 复制对象在指定帧上的全部变换通道。`通道` 表示粘贴属性通道。输出口未连接时生成 JSON，输出口连接时传递帧数据用于粘贴。 |

## 第一批通用输入节点

| 节点 | 类型 | 输入 | 输出 | 作用 |
| --- | --- | --- | --- | --- |
| 当前帧 | 数据获取 | `时间: NUMBER`，可选 | `帧: FRAME_LIST` | 输入没连接时输出 Maya 当前帧；输入连接时输出指定数值帧。 |
| 范围帧 | 数据获取 | `起始帧: NUMBER`，可选；`结束帧: NUMBER`，可选 | `帧范围: FRAME_RANGE`、`帧: FRAME_LIST` | 起始/结束没连接时输出当前时间轴范围；连接后输出指定范围。 |
| 逐帧采样 | 数据转换 | `帧范围: FRAME_RANGE` | `帧列表: FRAME_LIST` | 默认逐帧处理。 |
| 获取对象关键帧 | 数据获取 | `节点: NODE_LIST`、`通道: CHANNEL_LIST` | `帧列表: FRAME_LIST` | 只处理已有关键帧时使用。 |
| 帧范围过滤 | 数据转换 | `帧列表: FRAME_LIST`、`帧范围: FRAME_RANGE` | `帧列表: FRAME_LIST` | 把关键帧限制在指定帧段内。 |
| 变换通道 | 数据转换 | `位移: BOOL`、`旋转: BOOL`、`缩放: BOOL` | `通道: CHANNEL_LIST` | 显式指定处理哪些变换通道。 |
| 获取通道栏选择 | 数据获取 | 无 | `通道: CHANNEL_LIST` | 使用 Channel Box 当前选择作为通道输入。 |
| 按名称排序节点 | 数据转换 | `节点: NODE_LIST` | `节点: NODE_LIST` | 改变列表顺序，从而改变 1 对 1 粘贴对应关系。 |
| 反转节点列表 | 数据转换 | `节点: NODE_LIST` | `节点: NODE_LIST` | 反转列表顺序。 |
| 按层级排序节点 | 数据转换 | `节点: NODE_LIST` | `节点: NODE_LIST` | 多个父子对象处理时父级优先。 |

## 第一批实现建议

先实现能搭出“复制粘贴世界坐标”的最小闭环：

核心功能节点：

1. 复制帧

通用输入节点：

1. 当前帧
2. 范围帧
3. 变换通道
4. 获取通道栏选择
5. 按名称排序节点
6. 反转节点列表
7. 按层级排序节点

这套完成后，用户可以通过不同端口连接得到：

- 单对象复制帧。
- 多对象复制帧。
- 当前帧复制。
- 时间范围逐帧复制。
- 输出未连接时生成 JSON。
- 输出连接时粘贴到连接对象属性。
- 只粘贴位移或只粘贴旋转。
- 通过排序节点调整 1 对 1 粘贴顺序。

第二批再补关键帧模式和更高级的对象判断：

1. 获取对象关键帧
2. 帧范围过滤
3. 保存 / 读取帧数据文件
